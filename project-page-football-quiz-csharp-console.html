About me


2024 Women's Football Quiz

I wanted to make a small program that would help me practice some conditional statements and to get a program working from start to finish in the console. For this project I also wanted it to be about a subject I loved so I could showcase that in my Portfolio.

The quiz itself is simple in its execution and uses switch statements for the corresponding numbers in the grid. The grid resembles a Tic Tac Toe board. When a player answers a question correctly a '✓' mark will appear in the place of the corresponding number. If a player answers incorrectly a 'X' will appear.

After the first iteration I had a working program with some minor style elements I wanted to change. The first iteration of the program answer-block looked like this:

There is a lot of code that is repeated in the code block. For the second iteration I ran through the program and replaced code that is repeated multiple times with a method. The code looks much cleaner and is much more readable.

I'm happy with the results of my first complete console application. However. there is room for improvement. For instance, I want to change the answer input of the user to lower case. This gives much more flexibility when it comes to the answers of the player. Another very import aspect I need to look at is how to handle a null input. Even though it does not lead to a compiling error and the player can play the game none the less, I feel that it an improvement to have a method for a null input in place in the future, as this could handle potential exceptions and clean repeated code.
